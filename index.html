SQL

  1. SQL, or Structured Query Language, is a domain-specific programming language used for managing and manipulating relational databases.

  2. It is commonly used in conjunction with relational database management systems (RDBMS) such as MySQL, PostgreSQL, Microsoft SQL Server, SQLite, and Oracle Database.

DDL

DDL (Data Definition Language):

  1. CREATE: Used to create database objects like tables, views, indexes, etc.
    2. CREATE TABLE employees ( employee_id INT PRIMARY KEY, first_name VARCHAR(50), last_name VARCHAR(50), salary DECIMAL(10, 2) );

  2. ALTER: Modifies the structure of existing database objects (e.g., adding a column to a table).

    3. ALTER TABLE employees ADD COLUMN email VARCHAR(100);
  3. DROP: Deletes database objects (e.g., tables, views, indexes).
    4. DROP TABLE employees;
  4. TRUNCATE: Removes all records from a table, but retains the table structure for future use.
DML

DML (Data Manipulation Language):

  1. INSERT: Adds new records to a table.
    2. INSERT INTO employees (employee_id, first_name, last_name, salary) VALUES (1, 'John', 'Doe', 50000);
  2. UPDATE: Modifies existing records in a table.
    3. UPDATE employees SET salary = 55000 WHERE employee_id = 1;
  3. DELETE: Removes records from a table.

    4. DELETE FROM employees WHERE employee_id = 1;
DQL

DQL (Data Query Language):

    SELECT: Primarily used for querying the database to retrieve specific information. Focuses on retrieving and presenting data without modifying the database structure.

    SELECT column1, column2, ... FROM table_name WHERE condition;
DCL

    DCL (Data Control Language): GRANT: Provides specific privileges to users or roles. GRANT SELECT, INSERT ON employees TO username; REVOKE: Removes specific privileges from users or roles. Manages access to the database by controlling who can perform specific operations.

    GRANT SELECT, INSERT ON employees TO username;
